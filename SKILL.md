# Analyze

Read **all** of `me/context/` + `me/journal/` to date and produce a to-date (TD) analysis — a `YYYY-MM-DD TD analysis.md` file in `me/journal/`. The goal: spot patterns the human can't see from inside, challenge their assumptions usefully, and ask the questions that move them forward.

Read `CLAUDE.md` first. **Hard rule: never edit any `journal/` entry or `context/` file. You only read them and write the new TD analysis file.** Ignore `me/classes/` and `me/work/`.

## Inputs

1. **`me/context/`** — durable facts: who they are, situation, values, interests. Ground every observation here so your read is about *this* person, not a generic one.
2. **All `me/journal/` entries to date** — daily notes, in chronological order. Watch the arc across time, not just the latest entry.
3. **Prior TD analyses** — read the most recent one(s) so you build on them and can note what's changed, what resolved, and what recurred.

## How to question — the analytical engine

This is the heart of the skill. Apply these techniques; aim to provoke insight, not to comfort.

- **Socratic questioning** — don't assert conclusions, interrogate them. Probe assumptions ("what has to be true for that to hold?"), evidence ("what's the actual support for this belief?"), alternative viewpoints ("how would someone who disagreed see it?"), and implications ("if that's true, what follows?").
- **Downward-arrow technique** — when you hit a charged belief ("I have to take this job"), keep asking "and if that were true / didn't happen, what would that *mean*?" until you reach the core belief underneath. Surface the bottom of the chain.
- **Pattern recognition across time** — name recurring themes, words, moods, avoidances, and contradictions that span multiple entries. Quote the entries. Flag where stated values and actual choices diverge. Note what they keep circling back to but never resolve.
- **Cognitive restructuring / reframing** — where you spot a likely distortion (catastrophizing, black-and-white thinking, mind-reading, "shoulds"), name it gently and offer a more accurate, more useful framing — as a proposal to test, not a verdict.
- **Challenge, don't flatter** — the value of this analysis is the friction. Push on evasions and convenient stories. Be direct and specific, grounded in their own words.
- **End on open questions** — close with a short set of sharp, genuinely open questions for them to sit with. Open, not leading; provocative, not rhetorical. These are the deliverable they'll act on.

## Output

Write `me/journal/<YYYY-MM-DD> TD analysis.md`:

- Start with a `linked to:` line naming the context files and journal entries it draws on.
- Structure it (suggested): **Patterns I'm seeing** → **Tensions & contradictions** → **Assumptions worth challenging** → **Reframes to test** → **Open questions**.
- Be deep and structured; quote the source entries; keep it about this specific person.
- Keep it honest and challenging — this is a thinking partner's read, not a pep talk.

After writing, give the human a brief pointer to the file and the 2–3 most important things you want them to actually sit with.
